# ダンススケジュールアプリ

作成日時: 2026年4月2日 10:32

# 目的

- Webアプリケーション開発を一から学ぶ
- ダンスサークルでの日程調整に時間がかかる課題を解決し、効率化を図る

# 前提状況

- サークルの人数は50~100人
- 一つのダンス公演のナンバー数が20前後
- 一つのナンバーには、振り付け者（1, 2人程度）がいる
- 一回のナンバーの練習は90分
- スケジュール係がいる
- 各メンバーは、複数個のナンバーに出演する
- 振り付け者は練習に必ず出席できなければいけない

# 課題

1. 各メンバーは1ヶ月に一度、1ヶ月分の練習に参加できる日時を出演しているそれぞれのナンバーで提出する
    - 複数回同じ日程を送信するのは無駄
2. それぞれの振り付け者は、そのナンバーの出演者全員の提出をまとめて、複数の欠席の少ない日時を候補として共通のエクセルに書き込むことでスケジュール係に提出する
    - 欠席の少ない日時を集計するのが大変
3. スケジュール係は各ナンバーの候補日程を見ながら、どのナンバーもできるだけ多くの出席できるような日程を決める
    - 各ナンバーの練習は、同時に開催もできるが、同時開催するナンバーの出演者が重複していない必要がある
    - 最適化問題で人がやるのは大変なのでできれば自動化したいが、練習場所の調整や出席回数のばらつきを抑えるなど考えることが多いので、ひとまずは手をつけない

# 要望

- まずは課題1, 2だけを解決したい
- 一つの公演と、振り付け者とナンバーと出演者を登録
- 出演者が空いている日時を提出
- 自動で各ナンバーの練習候補日時（欠席の少ない日時）を見られる
