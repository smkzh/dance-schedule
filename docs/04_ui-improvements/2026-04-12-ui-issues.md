# UI 改善メモ

作成日: 2026-04-12

---

## 0. トップページ — アプリの目的が伝わらない

**場所**: `/c/[circleId]/`  
**現状**: タイトルが「公演一覧」だけで、初めて訪れたユーザーには何のアプリかわからない  
**改善案**: アプリ名やサブタイトル（例：「ダンスサークル練習日程調整」）をヘッダーに追加する

---

## 1. 日程提出ページ — 日付フォーマット

**場所**: `/c/[circleId]/submit/[performanceId]` の時間選択セクション  
**現状**: 「2026-04-01 の空き時間を選んでください」（ISO フォーマット）  
**改善案**: 「4月1日の空き時間を選んでください」のように日本語フォーマットに変える

---

## 2. ガントチャート — カラー凡例がない

**場所**: `/c/[circleId]/numbers/[performanceId]/[numberId]`  
**現状**: セルの濃さで欠席人数を表現しているが、凡例がないため初見ユーザーには意味が伝わらない  
**改善案**: チャート付近に「薄い = 欠席少 / 濃い = 欠席多」のような凡例を追加する

---

## 3. 公演編集 — 「振付」ラベルが全出演者に表示される

**場所**: `/c/[circleId]/admin/[performanceId]` のナンバーセクション  
**現状**: 振付者でないメンバーにも「□ 振付」チェックボックスが並び、全員が振付者候補に見える  
**改善案**: チェックされている人だけバッジ表示にするなど、振付者であることを差分で示す

---

## 4. ガントチャート — 「候補日のみ表示」のスタイル

**場所**: `/c/[circleId]/numbers/[performanceId]/[numberId]` のヘッダー右側  
**現状**: リンク風（下線テキスト）のスタイルだが、クリックするとトグル動作をする  
**改善案**: チェックボックスやボタンなど、状態を持つインタラクションであることが伝わるスタイルにする
