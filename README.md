# dance-schedule

ダンスサークル向けのスケジュール調整 Web アプリ。

**URL: https://dance-schedule-nine.vercel.app/**

---

## 制作背景

所属するダンスサークルでは、複数のナンバーに出演するメンバーがそれぞれの振り付け者に個別に日程を提出する必要があり、何度も同じ作業が発生していた。また振り付け者側も、出演者全員の予定を手作業で集計してどの日時が欠席者が少ないかを判断するのが大変だった。一度日程を提出するだけで全ナンバーに反映され、候補日の集計も自動化できるアプリがあれば双方の負担をなくせると考え、実際に使うことを目的として開発した。

---

## 機能

- **日程提出**: メンバーが空き時間を 30 分単位で提出する
- **候補日確認**: ナンバーごとに練習候補日をガントチャートで表示する
- **公演管理**: 管理者が公演・メンバー・ナンバーを追加・編集・削除する
- **CSV アップロード**: Google スプレッドシートから出力した CSV で公演を一括登録する

## 使い方

### メンバー向け
1. トップページで公演を選んで「日程を提出する」をタップ
2. 名前を選んでカレンダーから日付を選択
3. 空いている時間帯をタップして「提出する」

### スケジュール係向け
1. トップページで公演を選んで「候補日を確認する」をタップ
2. ナンバーを選ぶと練習候補日がガントチャートで表示される

### 管理者向け
1. トップページ下部の「管理ページへ」をタップ
2. 公演の追加・編集・削除ができる
3. 公演編集ページでメンバー・ナンバー・出演者を管理できる
4. 「新しい公演をCSVで登録」から CSV をアップロードして公演を一括登録できる

## 工夫した点

**将来の複数サークル対応を見越した設計**

現時点では1サークルのみの利用だが、将来複数サークルへ拡張できるよう最初から `circles` テーブルを設け、URL を `/c/[circleId]/` の構造にした。スキーマ変更なしで複数サークルに対応できる設計になっている。

**ガントチャート UI**

ナンバーごとの練習候補日を、日付×時間帯のガントチャートで表示した。欠席者数をセルの色で表現し、月ナビゲーション・欠席閾値フィルター・全日付表示の切り替えを実装することで、スケジュール係が候補日を直感的に判断できるようにした。

---

## 今後の改善点

- **認証機能の追加**: ログイン機能を設けることで複数サークルへの対応、他人の日程を編集できないようにする制限、ロールによる公演編集権限の管理を実現する
- **スケジュール係向け機能の充実**: ナンバーの出欠状況一覧や練習候補日の自動提案など、スケジュール決定を補助する機能を追加する
- **UX の改善**: CSV アップロード時にどんなファイルを用意すればよいか・スプレッドシートからの書き出し方をモーダルで案内するなど、IT に詳しくないメンバーでも使いやすくする
- **公演のUI登録対応**: 現在は CSV アップロードのみで公演を新規登録できるが、UI上でも直接追加できるようにする

- **デプロイ**: Vercel

## ローカル開発

```bash
npm install
npm run dev   # http://localhost:3005
```

`.env.local` に以下を設定する：

```
NEXT_PUBLIC_SUPABASE_URL=...
NEXT_PUBLIC_SUPABASE_ANON_KEY=...
```
